Red Team vs Blue Team Security Simulation

Welcome to the interactive security testing environment. This platform demonstrates critical web application vulnerabilities and their corresponding defensive mitigations.

You can toggle between the Vulnerable Mode (Red Team) and the Secure Mode (Blue Team) using the controller in the navigation bar. Analyze how behavior changes under attack scenarios.

🟥 RED TEAM (Attacker Perspective)

Observe and exploit weaknesses within the application. Key areas to inspect:

  • SQL Injection: Inject payloads into the Catalog Search input to extract private database entries.
  • Cross-Site Scripting (XSS): Embed scripts inside the Feedback form to run arbitrary scripts.
  • IDOR: Manipulate user identifiers in the Profile URL to view accounts of other registered users.
  • Broken Authentication: Exploit missing rate limits or reverse weak password controls during login.
  • Security Misconfiguration: Inspect the public Debug Panel to extract server environmental secrets.
🟦 BLUE TEAM (Defender Perspective)

Verify patches and review mitigations implemented when security controls are active:

  • Prepared Statements: Prevents SQL injection attacks by binding query parameters.
  • Output Encoding: Escapes special tags in the template view block to render payloads as harmless text.
  • Access Validation: Enforces login checks and ownership matching before serving profile resources.
  • Bcrypt Password Hashing: Replaces weak checks with hashed password checks.
  • CSRF Token & Headers: Rejects automated form submissions and injects defensive HTTP security headers.
